# Project Instructions - Lounge Time

## 프로젝트 개요
**Lounge Time** - 직장인을 위한 3분 휴식 웹 애플리케이션
- 컨셉: 직장인이 업무 중 3분 동안 눈치 보지 않고 짧은 휴식을 즐길 수 있는 라운지 앱
- 사용자 경험: 아이스크림을 먹는 인터랙션을 통해 휴식 시간을 시각화
- 분위기: 테마와 오디오를 통해 온라인 라운지에 들어온 느낌 제공

## 기술 스택
- **프론트엔드**: React + TypeScript
- **스타일링**: CSS (순수 CSS, 외부 UI 라이브러리 금지)
- **배포**: Microsoft Azure
- **필수 조건**: 빌드 성공 필수

## 주요 기능

### 1. 아이스크림 인터랙션
- 화면 중앙에 큰 아이스크림 막대 이미지 표시
- 마우스/터치로 눌렀을 때 점진적으로 아이스크림이 줄어듦
- 진행율(%)이 시각화됨

### 2. 동적 UI 변화
- 진행율이 올라갈수록 화면 분위기가 점점 더 시원한 색상/효과로 변화
- 깔끔하고 심플한 디자인 유지
- 마치 실제 카페에 있는 느낌

### 3. 테마 시스템
- **산 카페 테마**: 산장의 따뜻한 카페 분위기
- **폭포 테마**: 폭포 주변의 시원하고 상큼한 분위기
- 테마 선택 기능 제공

### 4. 오디오 시스템
- **에어컨 소리**: 산뜻한 배경음
- **사람들 대화 소리**: 라운지 분위기
- **폭포 소리**: 자연스러운 휴식감
- 테마에 따라 오디오 조합 변경
- 실제 오디오 파일 사용 또는 웹 오디오 API로 생성

### 5. 익명 채팅 기능
- 실시간 서버 없이 구현
- 샘플 메시지: 미리 정의된 사용자들의 익명 메시지 표시
- 사용자 입력: 로컬에서만 메시지 입력 가능 (서버 저장 X)
- 온라인 라운지의 분위기 연출

### 6. 휴식 추적
- 아이스크림을 끝까지 먹으면 "휴식 완료" 카운트 증가
- 브라우저 로컬스토리지에 데이터 저장
- 사용자가 총 휴식 횟수 확인 가능

## 코딩 스타일
- 들여쓰기: 2칸 (React/TypeScript 표준)
- 세미콜론: 필수
- 주석: 명확하고 간결하게
- 함수 및 컴포넌트에는 JSDoc 주석 추가
- 타입 안정성: 모든 변수에 TypeScript 타입 지정

## 개발 가이드라인
1. 컴포넌트는 작고 재사용 가능하게 설계
2. 상태 관리: useState, useEffect 중심으로 구현
3. CSS는 컴포넌트별로 분리
4. 반응형 디자인 필수 (모바일 친화적)
5. 접근성 고려 (ARIA 속성 등)
